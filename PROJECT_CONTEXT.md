# Contexto do Projeto

Você atuará como desenvolvedor Front-End Senior neste projeto.

## Objetivo

Estamos desenvolvendo um portfólio profissional para um escritório de arquitetura.

O site deve transmitir sofisticação, minimalismo e modernidade, com foco na apresentação visual dos projetos.

A aplicação deve funcionar perfeitamente em Desktop, Tablet e Mobile, seguindo a abordagem Mobile First.

## Tecnologias

* React
* TypeScript
* Vite
* TailwindCSS
* React Router
* Framer Motion
* Lucide React

## Objetivos de desenvolvimento

Quero que todas as implementações priorizem:

* Código limpo
* Componentes reutilizáveis
* Tipagem completa
* Organização do projeto
* Boa experiência do usuário
* Performance
* Acessibilidade
* SEO

Nunca implemente soluções rápidas apenas para funcionar. Prefira soluções escaláveis.

## Estrutura do site

O site possuirá as seguintes páginas:

* Home
* Projetos
* Detalhes do Projeto
* Serviços
* Sobre
* Contato

## Home

A Home será composta pelas seguintes seções:

* Hero
* Sobre resumido
* Projetos em destaque
* Serviços
* Depoimentos
* Chamada para contato
* Footer

Cada seção deverá ser um componente independente.

## Página de Projetos

Será composta por:

* Grid responsivo
* Filtros por categoria
* Cards de projeto
* Navegação para página de detalhes

## Página de Detalhes

Cada projeto terá:

* Banner
* Informações do projeto
* Galeria de imagens
* Descrição
* Localização
* Área construída
* Ano

## Serviços

Cada serviço será apresentado em cards reutilizáveis.

## Contato

A página conterá:

* Informações de contato
* Redes sociais
* Formulário
* Localização

## Organização do código

Cada componente deve possuir sua própria pasta.

Exemplo:

components/
Navbar/
Hero/
Gallery/
Footer/
ProjectCard/

Cada componente deve conter:

* index.tsx
* types.ts
* styles (quando necessário)

Evite componentes muito grandes.

## Design

O design deve ser elegante e minimalista.

Priorizar:

* Muito espaço em branco
* Fotos grandes
* Pouco texto
* Animações suaves
* Tipografia refinada

Não utilizar cores muito fortes.

## Desenvolvimento

Antes de implementar qualquer funcionalidade:

1. Analise a estrutura existente.
2. Explique rapidamente o plano de implementação.
3. Reutilize componentes sempre que possível.
4. Caso exista mais de uma solução, escolha a mais escalável.
5. Não altere componentes já existentes sem necessidade.
6. Caso seja necessário alterar arquitetura, explique o motivo.

Sempre mantenha consistência em todo o projeto.
